8/** \file kos/rwsem.h
9 \brief Definition for a reader/writer semaphore.
10 \ingroup kthreads
11
12 This file defines a concept of reader/writer semaphores. Basically, this
13 type of lock allows an unlimited number of "readers" to acquire the lock at
14 a time, but only one "writer" (and only if no readers hold the lock).
15 Readers, by definition, should not change any global data (since they are
16 defined to only be reading), and since this is the case it is safe to allow
17 multiple readers to access global data that is shared amongst threads.
18 Writers on the other hand require exclusive access since they will be
19 changing global data in the critical section, and they cannot share with
20 a reader either (since the reader might attempt to read while the writer is
21 changing data).
22
23 \author Lawrence Sebald
24*/
25

253/** \brief Upgrade a thread from reader status to writer status (with a
254 timeout).
255
256 This function will upgrade the lock on the calling thread from a reader
257 state to a writer state. If it cannot do this at the moment, it will block
258 until it is possible. This function is <b>NOT</b> safe to call inside an
259 interrupt.
260
261 You can only have one reader waiting to upgrade at a time, otherwise the
262 state would potentially become corrupted between when this is called and
263 when you get the lock. If you get -1 back from this, you must not assume
264 that you can write safely! On error, the calling thread will still hold a
265 read lock.
266
267 \param s The r/w semaphore to upgrade.
268 \param timeout The maximum time to wait (in milliseconds).
269 \retval 0 On success.
270 \retval -1 On error, errno will be set as appropriate.
271
272 \par Error Conditions:
273 \em EPERM - called inside an interrupt \n
274 \em EINVAL - the semaphore is not initialized \n
275 \em EINVAL - the timeout value is invalid \n
276 \em EBUSY - another reader has already requested an upgrade \n
277 \em ETIMEDOUT - the timeout expired before the write lock could be
278 acquired
279*/
280int rwsem_read_upgrade_timed(rw_semaphore_t *s, int timeout);
